## What this is (from the old site, in your own words)

The post is a walk-through of the Deck-to-Storyboard workflow. The version of this you wrote up before went something like:

> "When the idea is already approved and sitting in a slide deck, this workflow turns those slides into a full visual response: storyboards, hero frames, and variations, without you manually rewriting anything."

It runs in four phases.

### Phase 1, Drop In the Deck

- **Step 1, upload the slides.** Screenshots or copied slides from the approved presentation.
- **Step 2, add optional references.** Reference images (locations, products, mood shots) to keep everything grounded in reality.

### Phase 2, AI Reads the Slides

- **Step 1, understand the content.** The system reads headings, bullets, diagrams, emphasis points.
- **Step 2, turn it into a visual brief.** That slide becomes a short visual brief: what to show, mood, style, focus.

### Phase 3, Turn Slides into Shots

- **Step 1, define key shots.** For each slide, design a small set of shots: supporting angles plus a main hero image that "answers" the slide.
- **Step 2, write image directions.** Clear, natural-language directions that can be used to generate or edit images.
- **Step 3, create variations.** Multiple variations per shot (different angles, crops, moments) so you get coverage, not just a single frame.

### Phase 4, Storyboard-Ready Output

Three things drop out the other end:

- Frames you can drop straight into a storyboard.
- Images you can paste back into the deck.
- Options you can show the client as a visual response to their approved idea.

That's the workflow.

## Why this is worth a post

Every brand team has a deck. Most of them have multiple decks. Decks become work, work becomes assets, assets become campaigns. Today, that whole chain is human-bottlenecked: someone reads the deck, someone briefs the design team, someone briefs the production team, someone makes the visuals, someone reviews them against the deck.

This workflow collapses the chain. Approved deck in, storyboard out. The post should make that obvious in the first ten seconds.

The Slide Example folder is the proof: an actual slide input alongside six different visual outputs the system produced from it. That's the demo. Don't bury it.
