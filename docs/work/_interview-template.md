# Interview Template

When Zein names a new post, Claude walks through these questions one at a time. Answers go into `posts/<slug>.md`. Goal: capture everything needed so the eventual page can be one-shotted later.

Don't ask all questions at once, go conversational. Fill in what's already known, ask the rest.

---

## A. The story

1. **In one sentence, what is this post about?**
2. **Who is it for / what client?** (or "topic post, no client")
3. **What's the format / scope?** (e.g. "4-min film", "internal tool", "spec ad", "explainer about X")
4. **When did it happen / how long did it take?**

## B. Why it's worth a post

5. **What was the situation BEFORE?** What's the status quo this challenges or replaces?
6. **What did you actually do?** Walk me through the approach, process, key decisions, anything custom you built.
7. **What's the WOW?** Speed, quality, scale, originality, the one thing that makes someone stop scrolling.
8. **What proof do we have?** Final piece, BTS, screens, sample outputs, before/after, data, testimonials.

## C. The reader

9. **Who's this aimed at?** Brand teams? Agencies? Founders? Other operators?
10. **What's the takeaway you want them to leave with?**
11. **What action do you want them to take?** (book a call, see other work, share, etc.)

## D. Production

12. **Loom intro**, recorded? Where's the link?
13. **Hero asset**, what's the main visual at the top of the page (after the Loom)?
14. **Animated graphics**, what concept on this page would benefit from a small animated explainer? (e.g. a process diagram, a before/after slider, a tool demo loop)
15. **Other assets**, list everything we have (films, stills, screens, decks, BTS, etc.).

---

## How to use this

Open `posts/<slug>.md`. Each section maps to one of A–D above. Claude asks question by question, writes Zein's answers verbatim under each question. When all are answered, the concept doc is **Draft** status and ready to be turned into a real page.
